# Quick Task 260420-a01 Plan

## Objective

Upgrade Android SOS alerts to the strongest realistic third-party behavior available in this Flutter codebase, including DND-policy access guidance, stronger Android notification channel settings, and full-screen SOS escalation where Android permits it.

## Tasks

### Task 1: Add Android SOS escalation bridge and channel management

- Add Android manifest/activity entries required for full-screen notifications and notification actions.
- Extend Android native code to:
  - report whether Notification Policy Access is granted
  - open the DND policy settings screen
  - create/update a new SOS notification channel version with bypass-DND when permitted
- Extend Dart SOS services and permission services to use that bridge without breaking the current public flow.

### Task 2: Move Android SOS rendering onto the stronger local escalation path

- Update SOS backend push composition so Android can use a local urgent-alert rendering path while iOS remains unchanged.
- Update `main.dart` background FCM handler and `SosNotificationService` so Android SOS alerts can render full-screen local notifications in both foreground and background isolates.
- Keep fallback behavior safe and explicit when DND/full-screen permissions are missing.

### Task 3: Update honest user copy, docs, and validation coverage

- Update SOS/help/onboarding copy so Android escalation is described as best-effort, permission-aware behavior.
- Add concise code comments and quick-task summary documentation for testing and platform limits.
- Run focused Flutter and Node validation covering the changed SOS path.
